## 新增文档规则

- 生产事故、用户投诉、反复踩坑必须补文档。
- 标题建议使用递增编号 + 主题 + 日期，便于排序。
- 不确定原因时不要写成定论，应写“可能原因 / 验证方式 / 排除依据”。
- 如果文档改变系统运行理解，应同步更新 `00_文档索引与系统架构总览_20260714.md`。
- 如果 README 出现固定版本号，发版前必须检查是否过时。
- 涉及 Lite / Pro、拆仓、商业化、auto router 护城河判断时，必须先读 `32_产品定位与仓库策略_20260718.md`。

## 发布前文档检查

```bash
rg -n "v0\\.2\\.[0-9]+|TODO|待补|待完善|以后补|暂未|需要补充" README.md docs/*.md
```

命中不一定都是错误；历史记录标题可以保留，但用户操作建议不应指向过时版本。
